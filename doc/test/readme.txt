### 测试方案

1. 黑盒  
- 条件覆盖测试  
目的：希望能覆盖到本项目的各种存储过程使用情况，保证不出问题。  
 i. 设计完备一点的测试用例（存储过程），可通过python工具发起查询
 ii. 环境尽可能贴近实际使用环境，要通过脚本使用dbaclient来发起查询
没有参数的存储过程调用，

- 压力测试
高并发测试，同时上线下线，大量的同时读写数据库。这样的压力条件下，存储是否正常。

- 随机测试
玩游戏。定时执行脚本查询日志文件中的错误。

- 针对性的性能测试
目的：研究和学习，给出这个dbx的使用参数
在不同压力条件下每个查询请求的处理返回时间
缓存

2. 白盒  
单元测试
